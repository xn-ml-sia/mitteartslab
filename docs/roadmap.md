# Generative Rock Roadmap

## Goal

Move from concept documentation to a prototype-grade product that validates user resonance and generation quality.

## Phase 0: Foundation (Documentation and Alignment)

Completed in this repository:

- research and cultural framing
- brand narrative and voice
- UX flows and component definitions
- technical architecture and API contracts

## Phase 1: Prototype Build

### Scope

- implement prompt-based multi-rock generation first (Use Case 2)
- implement single-rock reveal flow second (Use Case 1)
- ship private demo environment

### Deliverables

- front-end MVP with core screens
- backend generation pipeline with asynchronous jobs
- baseline moderation and logging

### Validation questions

- Do users feel emotional resonance with generated results?
- Does the concept reduce friction for expressing difficult feelings?
- Are generated sets diverse and non-repetitive?

## Phase 2: Quality and Safety Hardening

### Scope

- improve visual coherence and prompt adherence
- improve message quality and deduplication
- tighten moderation for harmful or culturally unsafe prompts

### Deliverables

- ranking and diversity tuning
- observability dashboards
- safety policy updates

## Phase 3: Shareability and Retention

### Scope

- richer export and sharing formats
- saved collections and favorites
- lightweight onboarding and examples

### Deliverables

- export variants (single card, strip, set summary)
- collection management
- improved first-run success rates

## Metrics by phase

### Phase 1 metrics

- generation completion rate
- reveal completion rate
- first-session save rate

### Phase 2 metrics

- duplicate output rate
- moderation intervention rate
- user-rated relevance

### Phase 3 metrics

- share rate
- returning user rate
- collection creation rate
